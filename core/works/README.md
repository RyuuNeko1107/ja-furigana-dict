# core/works — 作品単位辞書

特定の作品 (アニメ / ゲーム / ライトノベル / 漫画) の固有名詞・造語を
**1 作品 = 1 ファイル** で集めるディレクトリ。
ja-furigana lib の loader が **全階層再帰** で `*.toml` を拾うため、
`core/works/<medium>/<title>.toml` の構造で配置すればそのまま jukugo
(≥2 字 surface 固定読み) として lookup される。

## なぜ jukugo と分けるのか

- 一般語 (`core/jukugo/<genre>/<file>.toml` 等) と作品造語が混ざると
  「同じ表層が違う読み」のとき commit log で衝突が見えにくくなる
- 作品ごとに公式読みのソース (公式サイト / 作者監修書籍) が違うので、
  **ファイル先頭 comment に出典を書く** 運用にしたい
- 「自分の好きな作品だけロードしたい」需要が出たら、将来の loader 拡張で
  `load_filter: ["works/game/touhou", "works/anime/madoka"]` 等で対応しやすい

## サブポリシー (works/ の追加で必須)

`core/jukugo/*` の seed 方針は「現代の私人 / 私企業 / アニメ作品 / 商標は
seed しない (誤読リスク回避)」だった。 works/ ではこれを部分的に緩和する代わりに、
以下のサブルールを **必ず** 守る:

1. **原則は公式読み**
   - 作者公式 / 公式サイト / 公式書籍 / 公式 wiki に明記された読みを最優先
   - ファン推測 / 同人解釈 / 派生作品の独自設定は **非推奨**。 ただし、
     **広く一般的な通称** として定着している場合は採録可 (公式が明示しない通称が
     コミュニティで圧倒的多数派になっているケース)。 PR 説明にその旨を明記する
2. **ファイル先頭に出典を必ず書く**
   - 例: `# 出典: 上海アリス幻樂団 公式 (https://...)`、`# 出典: TVアニメ公式設定資料集`
   - 一般通称を採録する場合も、 「○○ wiki 等で広く使われる通称」 のように出典を書く
3. **古典的読みは現代読みが無い場合のみ** (現代口頭 = TTS 用途を想定)
   - 現代でも読まれる surface に対して古典読みが採録されていると、 現代の利用者は
     聞き取れない / 読めない問題が発生する
   - 古典作品の登場人物名等で **現代の定着読みが存在しない** 場合のみ古典定本ベース
4. **商標問題のあるエントリは除外**
   - 作品タイトルや主要ブランド名そのものは引用範囲だが、 商標的にグレーな
     コラボ商品名 / 二次商標は入れない
5. **1 作品 = 1 ファイル**
   - `core/works/<medium>/<title>.toml` の構造を守る
   - 作品が大きくて分けたい場合は `core/works/<medium>/<title>/{characters,locations,terms}.toml`
     のように 1 階層下げる (loader は再帰なのでロードされる)
6. **TOML 形式は jukugo と同じ**
   ```toml
   [meta]
   role = "works"
   description = "<作品名>: <概要>"

   [entries]
   "博麗霊夢" = "ハクレイレイム"
   ```
7. **単漢字 (1 文字) は jukugo と同様に絶対に入れない**
   - validate.py が cross-file 重複として検出して fail させる
8. **`core/jukugo/*` (一般読み) との競合時は一般読み優先**
   - 同 surface が works と jukugo (specialized 等) の両方に登録されると lib 側 jukugo
     loader で merge 時に後勝ちになる (genre dir 名 sort 順、 works が後)
   - 競合に気付いた maintainer は **一般的な読みを残し、 works 側を作品固有読みのまま
     残置 / 削除** を判断 (自動振り分けはしない)

## 追加要望

PR 歓迎。新規作品を入れる場合は:
1. 上記サブポリシー全部に該当するか確認
2. ファイル先頭に出典 URL を `# 出典: ...` 形式の comment で書く
3. ファイル冒頭に `[meta] description = "<作品名>: <概要>"` を追加
   (STATS.md の用途列に `tools/regen_stats.py` が自動取り込み)
4. PR description に「公式読みのソース」を明記
